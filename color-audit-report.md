## 7. Notes

* **Exact duplicate count** means opaque 6-digit HEX bases that appear as more than one distinct `#тАж` literal (e.g. shorthand vs long form). **Two** inventory bases (`#FFFFFF`, `#000000`) also mix HEX and `rgb`/`rgba` literals for the same normalized RGB; see groups 12тАУ13 in section 4.
* **Near-duplicate groups** use Euclidean RGB distance тЙд 25 on pairs, then **transitive grouping** (connected components), so a group can chain many hues if each step is within threshold; interpret as тАЬsimilarity network,тАЭ not тАЬall pairs mutually similar.тАЭ
* **`var(--тАж)` matches are not all colors** (e.g. `var(--spacing)`, shadow stack variables); they are included because they match the CSS-variable pattern requested.
* **Some `HSL:` inventory keys are truncated or malformed** where `hsl(` spans `var(` without a closing `)` on the same substring; **460** additional matches were classified as HSL/HSLA at scan time (not listed separately in section 6).
* **Some `TW:` rows are non-color utilities** (e.g. `text-sm`, `text-left`) because the Tailwind pattern is intentionally broad; treat those rows as pattern-noise when filtering for palette work.
* **Detection limits**: Class names built by string concatenation, template literals split across lines, or runtime-only values are not reliably detected.
* **Tailwind matching** is regex-based; some valid utilities may be missed or rare false positives may occur (`shadow-*` captures non-color tokens).
* **Context** (text/background/border/shadow) is a heuristic from a short surrounding slice; `unknown` is common in minified JS.
* **HSL** rows use the raw function text as the inventory key (`HSL:тАж`); they were not converted to HEX in this report.
* **RGBA** entries use keys like `#RRGGBB ╬▒=0.5` to separate alpha variants without implying blending onto a specific backdrop.
* **Scan roots**: `app/`, `components/`, `lib/`, `echly-extension/`, `public/`, `utils/`, `scripts/`, plus `tailwind.config.ts`, `middleware.ts`, `next.config.ts`. No separate top-level `src/` or `pages/` directory was present; `styles/` was not a top-level folder (global CSS lives under `app/`).
